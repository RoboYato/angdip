# LMS (Learning Management System)

Полнофункциональная система управления обучением с поддержкой ролевой модели, конфиденциальности и логирования для дипломной работы.

## Возможности

### Админ-интерфейс
- ✅ Управление курсами (создание, редактирование, удаление)
- ✅ Управление материалами (с поддержкой файлов)
- ✅ Управление ролями и уровнями доступа
- ✅ Управление пользователями
- ✅ Назначение ролей пользователям
- ✅ Логирование действий и аудит
- ✅ Тестирование (поддержка разных типов вопросов)

### Пользовательский интерфейс
- ✅ Просмотр доступных курсов по ролям
- ✅ Просмотр материалов курса
- ✅ Отслеживание прогресса обучения
- ✅ Загрузка и просмотр материалов
- ✅ Прохождение тестов

## Быстрый старт

### Требования
- Docker и Docker Compose
- или Node.js 18+, PostgreSQL 12+

### Запуск с Docker (рекомендуется)

1. **Клонируйте репозиторий**
```bash
cd angdip
```

2. **Запустите Docker Compose**
```bash
docker-compose up -d
```

3. **Проверьте статус сервисов**
```bash
docker-compose ps
```

4. **Откройте приложение**
   - Приложение: http://localhost:8888
   - Backend API: http://localhost:8888/api/health
   - Database: localhost:5432

5. **Заполнение БД тестовыми данными (курсы, документация, пользователи)**  
   Если нужны тестовые пользователи и данные, выполните seed в контейнере backend:
   ```bash
   docker compose exec backend sh -c "cd /app && npx ts-node src/db/seed.ts"
   ```
   Данные пользователей см. в разделе «Данные пользователей (после seed)» ниже.

> **Важно**: Все API-запросы идут через nginx на порт 8888; backend на 3000 извне недоступен.

### Запуск локально

#### Backend

1. **Установите зависимости**
```bash
cd backend
npm install
```

2. **Создайте .env файл**
```bash
cp .env.example .env
```

3. **Запустите миграции БД**
```bash
npm run migrate
```

4. **Запустите backend**
```bash
npm run dev
```

Backend будет доступен на http://localhost:3000

#### Frontend

1. **Установите зависимости**
```bash
cd frontend
npm install
```

2. **Запустите сервер разработки**
```bash
npm start
```

Frontend будет доступен на http://localhost:4200

> **Примечание**: При использовании Docker Compose приложение доступно на http://localhost:8888

## API Endpoints

### Аутентификация
- `POST /api/auth/register` - Регистрация
- `POST /api/auth/login` - Вход
- `GET /api/auth/profile` - Профиль пользователя

### Курсы
- `GET /api/courses` - Получить доступные курсы
- `GET /api/courses/:id` - Получить курс с материалами
- `POST /api/courses` - Создать курс (admin)
- `PUT /api/courses/:id` - Обновить курс (admin)
- `DELETE /api/courses/:id` - Удалить курс (admin)

### Материалы
- `GET /api/materials/:id` - Получить материал
- `POST /api/materials` - Создать материал (admin)
- `PUT /api/materials/:id` - Обновить материал (admin)
- `DELETE /api/materials/:id` - Удалить материал (admin)
- `POST /api/materials/upload` - Загрузить файл (admin)

### Тесты
- `POST /api/tests` - Создать тест (admin)
- `POST /api/tests/question/add` - Добавить вопрос (admin)
- `POST /api/tests/submit` - Отправить ответы на тест
- `GET /api/tests/:testId/results` - Получить результаты теста

### Admin
- `GET /api/admin/roles` - Получить роли
- `POST /api/admin/roles` - Создать роль (admin)
- `GET /api/admin/users` - Получить пользователей (admin)
- `POST /api/admin/users` - Создать пользователя (admin)
- `GET /api/admin/access-levels` - Получить уровни доступа
- `GET /api/admin/audit-logs` - Получить логи аудита (admin)

## Функции безопасности

- **JWT аутентификация** - Все запросы требуют валидный токен
- **Хеширование паролей** - Использует bcryptjs
- **CORS** - Настроен на безопасное взаимодействие
- **Аудит логирования** - Логируется доступ к материалам с грифом
- **Ролевой контроль доступа** - RBAC для управления правами

## База данных

### Основные таблицы

- **users** - Пользователи системы
- **roles** - Роли пользователей
- **access_levels** - Уровни конфиденциальности
- **courses** - Курсы обучения
- **materials** - Учебные материалы
- **files** - Загруженные файлы
- **tests** - Тесты и вопросы
- **user_test_results** - Результаты тестов
- **audit_log** - Логи действий пользователей

## Развертывание на продакшене

1. **Обновите переменные окружения** в `.env` файлах
2. **Измените JWT_SECRET** на безопасный ключ
3. **Установите SSL сертификаты**
4. **Используйте reverse proxy** (nginx, traefik)
5. **Включите логирование** и мониторинг

## Поддерживаемые типы тестов

- **Одиночный выбор** (single_choice)
- **Множественный выбор** (multiple_choice)
- **Ввод текста** (text_input)

## Уровни конфиденциальности материалов

Система поддерживает различные уровни доступа к материалам:
- Public (публичный)
- Confidential (конфиденциальный)
- Secret (секретный)
- Top Secret (совершенно секретный)

Для материалов с грифом ведется логирование всех действий.

## Решение проблем

### Ошибка подключения к БД
```bash
# Проверьте переменные окружения
docker-compose logs postgres

# Убедитесь что postgres запущен
docker-compose ps
```

### Frontend не загружается
```bash
# Очистите кэш браузера
# Проверьте консоль браузера на ошибки
docker-compose logs frontend
```

### Ошибки при загрузке файлов
```bash
# Проверьте права на папку uploads
chmod -R 777 backend/uploads

# Проверьте размер файла (макс 50MB)
```
